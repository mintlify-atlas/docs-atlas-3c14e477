## Overview

The `authService` provides authentication functions using Supabase Auth. It handles user sign-in, sign-out, session management, and authentication state changes.

### Setup Requirements

<Steps>
  <Step title="Enable Email Auth">
    Navigate to **Authentication → Settings** in your Supabase dashboard and ensure Email provider is enabled.
  </Step>
  
  <Step title="Create Admin User">
    Go to **Authentication → Users** and click **"Invite User"** to create the admin account, or use:
    
    ```javascript
    supabase.auth.admin.createUser({
      email: 'admin@picanteria.com',
      password: 'secure-password',
      email_confirm: true
    });
    ```
  </Step>
</Steps>

## Functions

### signIn

Authenticates a user with email and password.

```javascript
import { signIn } from '../services/authService';

const user = await signIn('admin@picanteria.com', 'password123');
```

<ParamField path="email" type="string" required>
  User's email address
</ParamField>

<ParamField path="password" type="string" required>
  User's password
</ParamField>

#### Returns

<ResponseField name="user" type="object">
  Authenticated user object from Supabase Auth
  
  <Expandable title="User Object">
    <ResponseField name="id" type="uuid">
      User unique identifier
    </ResponseField>
    
    <ResponseField name="email" type="string">
      User's email address
    </ResponseField>
    
    <ResponseField name="role" type="string">
      User role (e.g., 'authenticated')
    </ResponseField>
    
    <ResponseField name="created_at" type="string">
      ISO timestamp of user creation
    </ResponseField>
    
    <ResponseField name="last_sign_in_at" type="string">
      ISO timestamp of last sign-in
    </ResponseField>
  </Expandable>
</ResponseField>

#### Behavior

- Uses Supabase's `signInWithPassword` method
- Creates a new session upon successful authentication
- Session is stored in browser's local storage by default

#### Error Handling

Throws error if:
- Invalid email format
- Incorrect password
- User doesn't exist
- User email not confirmed (if email confirmation required)
- Network error or Supabase unavailable

**Common error messages:**
- `"Invalid login credentials"` - Wrong email/password combination
- `"Email not confirmed"` - User hasn't verified their email

### signOut

Signs out the current user and invalidates their session.

```javascript
import { signOut } from '../services/authService';

await signOut();
```

#### Returns

No return value (void).

#### Behavior

- Invalidates the current session
- Clears session data from local storage
- Triggers `onAuthStateChange` listeners with `null` session

#### Error Handling

Throws Supabase error if sign-out fails (rare).

---

### getSession

Retrieves the current user session (useful for page reloads).

```javascript
import { getSession } from '../services/authService';

const session = await getSession();
if (session) {
  console.log('User is logged in:', session.user.email);
} else {
  console.log('No active session');
}
```

#### Returns

<ResponseField name="session" type="object | null">
  Current session object, or `null` if not authenticated
  
  <Expandable title="Session Object">
    <ResponseField name="user" type="object">
      User object (same structure as `signIn` return value)
    </ResponseField>
    
    <ResponseField name="access_token" type="string">
      JWT access token
    </ResponseField>
    
    <ResponseField name="refresh_token" type="string">
      Refresh token for session renewal
    </ResponseField>
    
    <ResponseField name="expires_at" type="number">
      Unix timestamp when session expires
    </ResponseField>
    
    <ResponseField name="expires_in" type="number">
      Seconds until session expiration
    </ResponseField>
  </Expandable>
</ResponseField>

#### Behavior

- Checks local storage for existing session
- Automatically refreshes expired sessions using refresh token
- Returns `null` if no valid session found

#### Error Handling

Throws Supabase error if session retrieval fails.

### onAuthStateChange

Subscribes to authentication state changes.

```javascript
import { onAuthStateChange } from '../services/authService';

const { data: { subscription } } = onAuthStateChange((session) => {
  if (session) {
    console.log('User signed in:', session.user.email);
  } else {
    console.log('User signed out');
  }
});

// Clean up subscription when component unmounts
return () => subscription.unsubscribe();
```

<ParamField path="callback" type="function" required>
  Function called when auth state changes
  
  <Expandable title="Callback Signature">
    ```typescript
    (session: Session | null) => void
    ```
    
    <ParamField path="session" type="object | null">
      Current session (same structure as `getSession` return value)
    </ParamField>
  </Expandable>
</ParamField>

#### Returns

<ResponseField name="subscription" type="object">
  Subscription object from Supabase
  
  <Expandable title="Properties">
    <ResponseField name="data" type="object">
      Contains the subscription object
      
      <Expandable title="data.subscription">
        <ResponseField name="unsubscribe" type="function">
          Call this to stop listening to auth changes
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

#### Behavior

- Callback fires immediately with current session state
- Callback fires on subsequent auth events:
  - `SIGNED_IN` - User successfully authenticated
  - `SIGNED_OUT` - User logged out
  - `TOKEN_REFRESHED` - Session token automatically renewed
  - `USER_UPDATED` - User metadata changed

#### Important Notes

<Warning>
Always unsubscribe when component unmounts to prevent memory leaks:

```javascript
const { data: { subscription } } = onAuthStateChange(callback);
return () => subscription.unsubscribe();
```
</Warning>

## Usage Examples

<CodeGroup>

```javascript Login Page
import { signIn } from '../services/authService';
import { useState } from 'react';

function LoginPage() {
  const [email, setEmail] = useState('');
  const [password, setPassword] = useState('');
  const [error, setError] = useState('');

  const handleLogin = async (e) => {
    e.preventDefault();
    setError('');
    
    try {
      const user = await signIn(email, password);
      console.log('Login successful:', user.email);
      // Redirect to dashboard
      window.location.href = '/dashboard';
    } catch (err) {
      setError(err.message);
    }
  };

  return (
    <form onSubmit={handleLogin}>
      <input 
        type="email" 
        value={email} 
        onChange={(e) => setEmail(e.target.value)}
        placeholder="Email"
        required
      />
      <input 
        type="password" 
        value={password} 
        onChange={(e) => setPassword(e.target.value)}
        placeholder="Password"
        required
      />
      {error && <p className="error">{error}</p>}
      <button type="submit">Sign In</button>
    </form>
  );
}
```

```javascript Auth Context
import { createContext, useContext, useEffect, useState } from 'react';
import { getSession, onAuthStateChange, signOut } from '../services/authService';

const AuthContext = createContext(null);

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    // Check for existing session on mount
    getSession()
      .then((session) => {
        setUser(session?.user ?? null);
        setLoading(false);
      })
      .catch(() => {
        setLoading(false);
      });

    // Listen for auth changes
    const { data: { subscription } } = onAuthStateChange((session) => {
      setUser(session?.user ?? null);
    });

    return () => subscription.unsubscribe();
  }, []);

  const logout = async () => {
    await signOut();
    // User state automatically updated via onAuthStateChange
  };

  return (
    <AuthContext.Provider value={{ user, loading, logout }}>
      {children}
    </AuthContext.Provider>
  );
}

export const useAuth = () => {
  const context = useContext(AuthContext);
  if (!context) throw new Error('useAuth must be used within AuthProvider');
  return context;
};
```

```javascript Protected Route
import { Navigate } from 'react-router-dom';
import { useAuth } from '../contexts/AuthContext';

function ProtectedRoute({ children }) {
  const { user, loading } = useAuth();

  if (loading) {
    return <div>Loading...</div>;
  }

  if (!user) {
    return <Navigate to="/login" replace />;
  }

  return children;
}

// Usage in router
<Routes>
  <Route path="/login" element={<LoginPage />} />
  <Route
    path="/dashboard"
    element={
      <ProtectedRoute>
        <DashboardPage />
      </ProtectedRoute>
    }
  />
</Routes>
```

```javascript Header with Logout
import { useAuth } from '../contexts/AuthContext';

function Header() {
  const { user, logout } = useAuth();

  const handleLogout = async () => {
    try {
      await logout();
      window.location.href = '/login';
    } catch (error) {
      console.error('Logout failed:', error);
    }
  };

  if (!user) return null;

  return (
    <header>
      <div>Welcome, {user.email}</div>
      <button onClick={handleLogout}>Sign Out</button>
    </header>
  );
}
```

```javascript Session Check
import { getSession } from '../services/authService';

// Check if user is authenticated before API call
async function fetchProtectedData() {
  const session = await getSession();
  
  if (!session) {
    throw new Error('Not authenticated');
  }

  // Use access token for authenticated requests
  const response = await fetch('/api/protected', {
    headers: {
      'Authorization': `Bearer ${session.access_token}`
    }
  });

  return response.json();
}
```

## Security Best Practices

<AccordionGroup>
  <Accordion title="Store sessions securely">
    Supabase automatically stores sessions in `localStorage` by default. For enhanced security, configure session storage in `createClient`:
    
    ```javascript
    import { createClient } from '@supabase/supabase-js';
    
    const supabase = createClient(url, key, {
      auth: {
        storage: window.localStorage, // or implement custom storage
        autoRefreshToken: true,
        persistSession: true,
        detectSessionInUrl: true
      }
    });
    ```
  </Accordion>
  
  <Accordion title="Handle token expiration">
    Sessions automatically refresh before expiration. Handle refresh failures:
    
    ```javascript
    onAuthStateChange((session) => {
      if (!session) {
        // Session expired or user logged out
        window.location.href = '/login';
      }
    });
    ```
  </Accordion>
  
  <Accordion title="Validate on server-side">
    Always validate JWT tokens on your backend:
    
    ```javascript
    // In your API route
    const token = req.headers.authorization?.replace('Bearer ', '');
    const { data: { user }, error } = await supabase.auth.getUser(token);
    
    if (error || !user) {
      return res.status(401).json({ error: 'Unauthorized' });
    }
    ```
  </Accordion>
</AccordionGroup>
