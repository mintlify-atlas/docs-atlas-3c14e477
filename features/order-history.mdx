## Overview

The Order History page provides a retrospective view of all finalized orders (completed and cancelled) with powerful filtering by time period and payment method. Orders are displayed in a Kanban-style layout for easy visual scanning.

## Time Period Filters

Filter orders by creation date with preset time ranges:

| Period | Date Range |
|--------|------------|
| **Todo** | All orders (no filter) |
| **Hoy** | Today from 00:00 to 23:59 |
| **Esta Semana** | Monday of current week through today |
| **Este Mes** | First day of current month through today |
| **Mes Pasado** | First to last day of previous month |

**Implementation:**

```javascript
export const getRangoFechas = (periodo) => {
  if (periodo === "Todo") return null; // No filter

  const hoy = startOf(new Date());
  const manana = new Date(hoy);
  manana.setDate(hoy.getDate() + 1);

  if (periodo === "Hoy") 
    return { desde: hoy, hasta: manana };

  if (periodo === "Esta Semana") {
    const lunes = new Date(hoy);
    lunes.setDate(hoy.getDate() - (hoy.getDay() === 0 ? 6 : hoy.getDay() - 1));
    return { desde: lunes, hasta: manana };
  }

  if (periodo === "Este Mes") 
    return { desde: new Date(hoy.getFullYear(), hoy.getMonth(), 1), hasta: manana };

  if (periodo === "Mes Pasado") 
    return {
      desde: new Date(hoy.getFullYear(), hoy.getMonth() - 1, 1),
      hasta: new Date(hoy.getFullYear(), hoy.getMonth(), 1),
    };
};
```

**Date filtering (client-side):**

```javascript
const pedidos = (() => {
  const rango = getRangoFechas(periodo);
  if (!rango) return todosLosPedidos; // "Todo" selected
  
  return todosLosPedidos.filter((p) => {
    const f = new Date(p.created_at);
    return f >= rango.desde && f < rango.hasta;
  });
})();
```

<Tip>
  Client-side filtering prevents timezone issues that can occur when filtering dates in UTC on the server.
</Tip>

**Server query with payment filter:**

```javascript
export const getPedidosHistorial = async ({ pagos = [] } = {}) => {
  let query = supabase
    .from("pedidos_picanteria")
    .select(CAMPOS)
    .in("estado_pedido", ["completado", "cancelado"])
    .order("created_at", { ascending: false });

  if (pagos.length > 0) {
    query = query.in("metodo_pago", pagos); // Filter by selected methods
  }

  const { data, error } = await query;
  if (error) throw error;
  return data ?? [];
};
```

<Note>
  Payment filtering is applied **server-side** (Supabase query), while date filtering is **client-side** (JavaScript filter).
</Note>

**Revenue calculation:**

```javascript
const totalIngresos = pedidos
  .filter((p) => p.estado_pedido === "completado")
  .reduce((acc, p) => acc + Number(p.total_final ?? p.total_estimado ?? 0), 0);
```

<Warning>
  Revenue only counts **completed** orders. Cancelled orders are excluded from the total even if they had a `total_estimado`.
</Warning>

## Data Loading

The `usePedidosHistorial` hook manages data fetching:

```javascript
export const usePedidosHistorial = (periodo, pagosActivos) => {
  const [todosLosPedidos, setTodosLosPedidos] = useState([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  const pagosKey = pagosActivos.join(",");

  const cargar = useCallback(async () => {
    try {
      setLoading(true);
      setError(null);
      
      // Fetch with payment filter (server-side)
      const data = await getPedidosHistorial({ 
        pagos: pagosKey ? pagosKey.split(",") : [] 
      });
      
      setTodosLosPedidos(data);
    } catch (err) {
      setError(err.message);
    } finally {
      setLoading(false);
    }
  }, [pagosKey]);

  useEffect(() => { cargar(); }, [cargar]);

  // Filter by period (client-side)
  const pedidos = /* ... date filtering ... */;

  // Group by column
  const porEstado = /* ... grouping logic ... */;

  // Calculate revenue
  const totalIngresos = /* ... sum of completed ... */;

  return { pedidos, porEstado, totalIngresos, loading, error, recargar: cargar };
};
```

**Re-fetch triggers:**
- Payment filter changes (`pagosKey` dependency)
- Manual refresh button click (`recargar()` function)

<Tip>
  The `pagosKey` string (`"Yape,Plin"`) is used as a dependency to avoid re-rendering loops from array reference changes.
</Tip>

## User Workflows

### Viewing Daily Revenue

1. Select **"Hoy"** from period dropdown
2. Look at **"Ingresos netos"** stat card
3. Scroll through **"Entregados"** and **"Recogidos"** columns
4. Click **"Limpiar"** if payment filters are active

### Finding Cancelled Orders

1. Select desired time period (e.g., "Esta Semana")
2. Scroll to **"Cancelados"** column (rightmost)
3. Review red-accented cards with cancelled item chips
4. Check cancellation reasons (if stored in notes field)

### Filtering by Payment Method

1. Click **"Filtros"** button (top right)
2. Select desired payment methods (e.g., Yape + Plin)
3. Red badge shows active filter count
4. Orders update to show only selected methods
5. Click **"Limpiar"** to reset filters

### Reviewing Last Month's Performance

1. Select **"Mes Pasado"** from period dropdown
2. Check **"Total pedidos"** and **"Ingresos netos"** cards
3. Compare **"Entregados"** (delivery) vs. **"Recogidos"** (pickup) counts
4. Review **"Cancelados"** rate

## Error Handling

If data loading fails, an error banner appears:

```javascript
{error && (
  <div className="bg-red-50 border border-red-200 text-red-700 rounded-lg px-4 py-3 mb-4">
    <span className="material-icons-round">error_outline</span>
    Error al cargar historial: {error}
    <button onClick={recargar} className="ml-auto underline">
      Reintentar
    </button>
  </div>
)}
```

**Common errors:**
- Network timeout
- Supabase connection failure
- Invalid date range
- Missing table permissions
